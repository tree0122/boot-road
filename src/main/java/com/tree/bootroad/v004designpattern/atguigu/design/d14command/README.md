# 命令模式
基本介绍:
- 软件设计中, 经常需要向某些对象发送请求,
- 不需要指定命令的接收者(执行者), 
- 不需要被请求的是哪个操作
- 只需程序运行时, 指定具体的请求接收者

原理图:
1. Invoker: 调用者
1. Receiver: 执行者, 具体执行请求的相关操作
1. Command: 抽象, 要执行的所有命令
1. ConcreteCommand: 命令的实现, 将执行者和命令绑定, 并执行

空命令: 没有然后命令, 不执行任何操作.

注意事项和细节:
1. 请求对象和执行请求的对象解耦
1. 容易设计成一个命令队列
1. 不足: 可能导致某些系统有过多的具体命令类, 增加系统复杂度
1. 空命令, 省去判断空的操作
1. 经典场景: 界面的按钮, 订单的撤销/恢复, 触发/反馈机制
